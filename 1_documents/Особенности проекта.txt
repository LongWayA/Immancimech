Некоторые осбенности проекта.

Стиль кода.

Интерфейс обозначается _I(interface) в конце.
Например: interface Background_I

Класс обозначается _C(class)
Например: class TileForBackground_C

Главный объект в модуле обозначается _R(root-корень) 
Например: let ai_R = {
Так получилось потому что если объект уникальный то для него класс не нужен, достаточно интерфейса. Класс использую только если в проекте есть несколько экземпляров, например существа.
А ключевой объект должен как то выделяться.

---------------------------------------


iniM() внутренняя инициализация модуля.
Нужен для того чтобы инициализировать то что можно сразу.

startM() внешний старт инициализации модуля.
Нужен для того чтобы инициализировать то что должно подождать данные других модулей.
